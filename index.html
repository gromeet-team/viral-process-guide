1
아이디 수급
바니 / 벤 ⚠️ 테더링 필수
▼
  • 바니 담당: USDT(테더) 코인 구매
  • 벤 담당: 텔레그램에서 실명계정 판매자 컨택 → 실명 아이디 구매
  • 구매한 계정은 실명계정 정리 시트에 즉시 기록
⚠️ 필수 환경: 테더링 사용
프록시 절대 금지 — 실명계정 영구정지 방지. Wi-Fi X, 데이터 테더링만.
2
키워드 선정
Google Sheets
▼
  • Google Sheets에서 키워드 리스트 확보 (PC/MB/전체 검색량 포함)
  • ★★ 최우선: 검색량 1,000 이상
  • ★ 우선: 검색량 500~999
  • 일반: 검색량 100~499
  • 브랜드·제품 카테고리 관련 키워드 필터링 후 진행
시트 컬럼 구조: keyword / PC검색량 / MB검색량 / 전체검색량 / 순위 / 담당자 / 단계 / 상위노출 여부
3
회원가입 · 등업 (Mode 4)
AI 원고 생성기 V4
▼
  • 도구: AI 원고 생성기 V4 (llm-generator, Python/PySide6 기반)
  • 실행: python main.py → Mode 4 선택
  • 실명 아이디로 발행 가능한 네이버 카페 회원가입
  • 글쓰기 권한(등업) 조건 충족까지 자동 처리
V4 구조: mode4/ 폴더가 회원가입·등업 담당. config/에서 계정 목록 설정.
4
AI 원고 생성 (Mode 1)
AI 원고 생성기 V4 100% 자동
▼
  • Google Sheets의 keyword:컬럼명 → AI 입력값으로 사용
  • AI 모델 선택: OpenAI / Anthropic / Google Gemini 중 택 1
  • 생성된 원고는 output:컬럼명에 자동 저장
  • 수동 수정 불필요 (100% 자동화)
컬럼 네이밍 규칙:
· keyword:변수명 → 프롬프트에 {{변수명}}으로 삽입됨
· output:결과명 → 해당 컬럼에 AI 결과 자동 기입
5
카페 자동 게시 (Mode 2)
Playwright 실명계정
▼
  • Playwright 브라우저 자동화 사용
  • 실명 아이디로 네이버 로그인 → 생성된 원고 자동 업로드
  • 카페별 게시판 설정은 config/에서 관리
⚠️ 테더링 환경 유지 — Mode 2 실행 중에도 테더링 필수.
6
자동 댓글 · 대댓글 (Mode 3)
비실명계정
▼
  • 비실명 계정으로 댓글 + 대댓글 자동 포스팅
  • 자연스러운 반응 시뮬레이션 (Mode 3 처리)
  • 실명계정과 구분하여 운영 — 별도 계정 풀 관리 필요
7
순위 확인
일 발행량 100개+ 도달 시 도입
▼
  • 현재 미운영 — 일 발행량 100개 이상 도달 시 도입 예정
  • 키워드별 상위노출 여부 트래킹
  • Google Sheets 날짜별 순위 컬럼에 결과 기록

1
키워드 및 검색량 추출
키워드 도구
▼
  • 네이버 검색광고 또는 키워드 분석 도구 활용
  • PC 검색량 / MB 검색량 / 전체 검색량 함께 수집
  • Google Sheets에 정리 후 우선순위 분류
2
비실명계정 아이디 수급
외부 업체
▼
  • 외부 업체 통해 비실명계정 구매
  • 예: 위너 마케팅, 바이럴김선생, 블로그 재료 상사 등
  • 구매 후 계정 상태(활성/휴면/정지) 확인 필수
3
카페 찾기
수동 조사
▼
  • 발행 가능한 네이버 카페 선별
  • 카페 활성도, 등업 조건, 게시판 현황 확인
  • 키워드 카테고리에 맞는 카페 우선 선택
4
카페 깨우기 (회원가입·등업)
회원가입 프로그램
▼
  • 회원가입 및 등업 프로그램 사용
  • 카페 회원가입 자동 처리
  • 글쓰기 등업 조건 충족까지 처리 (가입 후 댓글/게시글 작성 등)
5
프록시 설정
쿨아이피 / 하이프록시 / 하이아이피
▼
  • 프록시 서비스 결제 및 설정: 쿨아이피, 하이프록시, 하이아이피 등
  • 계정별 IP 분리 운영 — 동일 IP 사용 금지
  • 프록시 상태 주기적 점검 필요
신규 방식과 차이: 신규 방식(실명계정)은 프록시 금지. 기존 방식(비실명계정)은 프록시 필수.
6
n8n 원고 생성
n8n Google Sheets 연동
▼
  • 각 브랜드별 n8n 워크플로우 제작
  • 시트에 제목, 본문, 댓글 10개 자동 생성
  • Google Sheets와 연동하여 결과 자동 기입
  • 브랜드별로 별도 워크플로우 운영
7
원고 서포터즈 검토
알바 담당
▼
  • 원고 서포터즈 알바 고용
  • 원고 흐름 검토: AI 어색한 문장 수정
  • 이미지 수집: 원고 내 이미지 자리마다 각 3장씩 찾아오기
  • 검토 완료 원고만 다음 단계 진행
8
카페 포스팅
포스팅 프로그램
▼
  • 카페 포스팅 프로그램 활용해서 자동 업로드
  • 계정별 프록시 연결 상태 확인 후 실행
9
미완료 원고 관리
시트 관리
▼
  • 돌아가지 않은 원고: 원래 자리로 이동 (재시도 대기)
  • 완료된 원고: 그대로 유지 (단순 백업 목적)
  • 재사용이 아닌 백업·추적 용도
10
순위 체크
순위체크 프로그램
▼
  • 순서: 결과 저장 → 내보내기 → 순위체크 프로그램 실행 → 불러오기
  • 포스팅 완료된 키워드 대상으로 실행
  • 순위 결과는 시트에 기록
11
데일리 기록
벤 / 린
▼
  • 상위 노출 달성 여부 확인
  • 조회수 확인
  • 일별 데이터 기록 (Google Sheets 날짜별 컬럼)
신규 vs 기존 방식 비교
🆕 신규 방식 (실명계정)
  • 실명계정 텔레그램 구매
  • USDT 코인으로 결제
  • 테더링 필수, 프록시 금지
  • AI 원고 생성기 V4 (Python)
  • Mode 1~4 통합 자동화
  • 수동 원고 검토 없음 (100% 자동)
  • 신규 방식 순위 확인 미운영 (예정)
🕰️ 기존 방식 (비실명계정)
  • 외부 업체 비실명계정 구매
  • 프록시 필수 (쿨아이피 등)
  • n8n 워크플로우 원고 생성
  • 알바 원고 검토 + 이미지 수집
  • 카페 포스팅 프로그램
  • 순위체크 프로그램 별도 운영
  • 데일리 조회수/순위 기록
